Getting Started

Include the SDK

Add the SDK script to your HTML file:

HTML 
<script src="static/js/myowndb.js"></script>

Initialize the SDK

Create a new instance with your API key:

JavaScript 
// Initialize with your API key
const db = new MyOwnDB('your-api-key-here');

// Optional: Custom base URL (default: http://127.0.0.1:8000/api/v1)
const db = new MyOwnDB('api-key', 'https://your-server.com/api/v1');

Start Building!

Use the SDK methods to interact with your database:

JavaScript 
// Fetch all rows from 'todos' table
const { data, error } = await db.from('todos').select();

if (error) {
    console.error('Error:', error);
} else {
    console.log('Data:', data);
}

SDK Methods

db.tables() GET

Retrieve a list of all tables available with your API key.

JavaScript 
const { data, error } = await db.tables();

// Response: { data: [{ name: 'todos', row_count: 5 }, ...], error: null }
db.from(tableName)

Select a table to perform operations on. This method is chainable.

ParameterTypeDescription
tableNamestringName of the table to select
JavaScript 
// Chain with other methods
db.from('todos').select();
db.from('users').insert({ name: 'John' });
db.from(table).schema() GET

Get the schema (columns) of a table.

JavaScript 
const { data, error } = await db.from('todos').schema();

// Response: { columns: [{ name: 'id', type: 'INTEGER' }, { name: 'title', type: 'TEXT' }] }
db.from(table).select(limit?) GET

Fetch all rows from a table. Optionally limit results.

ParameterTypeDescription
limitnumber (optional)Maximum number of rows to return
JavaScript 
// Get all rows
const { data } = await db.from('todos').select();

// Get first 10 rows
const { data } = await db.from('todos').select(10);

// Response: [{ id: 1, title: 'Buy milk', done: false }, ...]
db.from(table).insert(data) POST

Insert a new row into the table.

ParameterTypeDescription
dataobjectObject with column names and values
JavaScript 
const { data, error } = await db.from('todos').insert({
    title: 'Buy groceries',
    description: 'Milk, eggs, bread',
    done: false
});

if (!error) {
    console.log('Created!', data);
}
db.from(table).update(id, data) PUT

Update an existing row by its ID.

ParameterTypeDescription
idnumberThe row ID to update
dataobjectObject with updated values
JavaScript 
const { error } = await db.from('todos').update(1, {
    title: 'Updated title',
    done: true
});

if (!error) {
    console.log('Updated successfully!');
}
db.from(table).delete(id) DELETE

Delete a row by its ID.

ParameterTypeDescription
idnumberThe row ID to delete
JavaScript 
const { error } = await db.from('todos').delete(5);

if (!error) {
    console.log('Deleted!');
}

REST API Reference

If you prefer to use the REST API directly without the SDK, here are the available endpoints:

Method Endpoint Description
GET /api/v1/tables/ List all tables
GET /api/v1/tables/{table}/ Get table schema
GET /api/v1/tables/{table}/rows/ Get all rows
POST /api/v1/tables/{table}/rows/ Create a new row
PUT /api/v1/tables/{table}/rows/{id}/ Update a row
DELETE /api/v1/tables/{table}/rows/{id}/ Delete a row

All API requests require an API key passed in the header:

HTTP Header

Complete Examples

üì± Complete CRUD Example

A full example showing all CRUD operations:

JavaScript 
// Initialize SDK
const db = new MyOwnDB('your-api-key');

// CREATE - Add a new todo
async function createTodo(title, description) {
    const { data, error } = await db.from('todos').insert({
        title: title,
        description: description,
        done: false
    });
    return { data, error };
}

// READ - Get all todos
async function getTodos() {
    const { data, error } = await db.from('todos').select();
    return data || [];
}

// UPDATE - Mark todo as done
async function markDone(id) {
    const { error } = await db.from('todos').update(id, { done: true });
    return !error;
}

// DELETE - Remove a todo
async function deleteTodo(id) {
    const { error } = await db.from('todos').delete(id);
    return !error;
}

üìä Fetch & Display Data

Load and render data from your database:

JavaScript 
const db = new MyOwnDB('your-api-key');

async function loadAndDisplay() {
    // Show loading state
    document.getElementById('list').innerHTML = 'Loading...';

    // Fetch data
    const { data, error } = await db.from('products').select();

    if (error) {
        document.getElementById('list').innerHTML = `Error: ${error}`;
        return;
    }

    // Render items
    const html = data.map(item => `
        <div class="item">
            <h3>${item.name}</h3>
            <p>$${item.price}</p>
        </div>
    `).join('');

    document.getElementById('list').innerHTML = html;
}

// Load on page ready
loadAndDisplay();

‚ö†Ô∏è Error Handling Patterns

Best practices for handling errors:

JavaScript 
const db = new MyOwnDB('your-api-key');

async function safeOperation() {
    try {
        // Every SDK method returns { data, error }
        const { data, error } = await db.from('todos').select();

        // Always check for errors first
        if (error) {
            console.error('API Error:', error);
            showErrorToast(error);
            return null;
        }

        // Data is guaranteed to be valid here
        return data;

    } catch (e) {
        // Handle network or unexpected errors
        console.error('Unexpected error:', e);
        showErrorToast('Network error. Please try again.');
        return null;
    }
}

// Common error messages:
// - "No table selected. Use .from('tableName') first."
// - "Request failed" (invalid API key or server error)
// - Network errors (server unreachable)

Response Format

üì¶ Standard Response Structure

All SDK methods return a consistent response format:

JavaScript 
// Success response
{
    data: [...],  // The returned data (array or object)
    error: null   // No error
}

// Error response
{
    data: null,            // No data on error
    error: "Error message"  // Description of error
}
